網頁閱讀最常見的誤區，是以為「讀了多少篇」代表「帶走多少東西」。真正決定閱讀品質的不是讀量，是一條從「訊號進來」到「知識留下」的完整流程——先決定什麼值得讀、再讓讀的當下門檻夠低、最後讓有價值的內容真的被保留下來。這篇分享我怎麼用工具組合把這條流程跑順。

我每天大概會接觸三、四十篇內容——技術文章、新聞、研究、推薦閱讀。以前的做法是看完就關掉，剩下的印象大概兩天就淡了。後來慢慢摸索出一套「擷取 → 閱讀 → 收藏」的三段流程：**擷取**段決定什麼訊號進來、**閱讀**段讓內容門檻降到最低、**收藏**段決定什麼留下來變成長期知識。這篇就是我現在的配置，工具從瀏覽器擴充到 app 都有。

## 擷取與降噪

第一段：決定什麼值得讀。大部分的閱讀內容品質問題，其實發生在「點進去之前」——訊號源頭選不好、演算法推的不是你真正想要的、RSS 訂閱沒整理。這一段的工具處理訊號入口，把雜訊濾在進入視線之前。

### Kagi News

<Image src={kagiNews} alt="Kagi News" />

Kagi News 是 [Kagi 搜尋引擎](https://kagi.com) 推出的新聞服務，用一組經過人工篩選的來源清單產生每日新聞摘要，主打「不靠演算法、不靠廣告、只看來源本身的品質」。對我來說它解決的是「日常新聞基礎訊號」的問題——不用刷推特或臉書就能掌握當天的主要消息，而且那些來源本身品質夠高，不用再多做一層過濾。

<span class="pricing-tag paid">需訂閱 Kagi</span>
<a href="https://kagi.com/news" target="_blank" class="official-link">官方網站</a>

### Folo

<Image src={folo} alt="Folo" />

[Folo](https://folo.is/) 是一個開源的 RSS reader，把各種來源——RSS、newsletter、YouTube 頻道、社群平台——整合到單一閱讀介面。它的設計讓你把「要看的人跟站」用訂閱的方式固定下來，而不是被演算法餵食。我用它處理更精準的個人興趣追蹤——技術部落格、特定創作者、Substack 訂閱——這些內容用演算法推薦幾乎接不到，靠 RSS 訂閱反而穩定。

<span class="pricing-tag freemium">免費使用，進階功能需訂閱</span>
<a href="https://folo.is/" target="_blank" class="official-link">官方網站</a>

### Grok Tasks

<Image src={grokTasks} alt="Grok Tasks" />

Grok Tasks 是 [xAI 的 Grok](https://grok.com) 提供的排程任務功能，可以預設 prompt 讓 Grok 每天自動執行一次。我的用法是讓它在每天清晨用 [Axios 新聞網](https://www.axios.com) 的 smart brevity 寫作風格整理昨日重要新聞——短句、分點、每條新聞先講結論再講 context，快速掃完就能對前一天的大事有基礎掌握。

為什麼用 Axios 風格而不是讓它寫成長文？因為 smart brevity 的設計就是為了「一眼能吸收」，跟晨間快速掌握資訊的場景完美契合。真的需要深入的再手動查。

Prompt 骨架大概像這樣：

<p class="code-title">Grok Task Prompt</p>

```
用 Axios 新聞網的 smart brevity 風格整理昨日全球重要新聞 5–7 條。每條格式：
1. 一句話標題
2. "Why it matters:" 點出這件事為什麼重要
3. "The big picture:" 給出一到兩句背景

全部用繁體中文輸出。
```

<span class="pricing-tag freemium">免費可用但有額度上限，完整功能需訂閱 SuperGrok 或 X Premium+</span>
<a href="https://grok.com" target="_blank" class="official-link">官方網站</a>

三個工具並用，基本上把「今天要讀什麼」這件事從演算法手上收回來：Kagi News 負責「廣度」（主流新聞全覽、人工篩選來源）、Folo 負責「深度」（我主動 curate 的個人興趣來源）、Grok Tasks 負責「精煉」（AI 自動摘要成 smart brevity 格式）。

## 閱讀

第二段：讀的當下把門檻降到最低。外文要翻譯、影片字幕要跟得上、中英混排要有空格。這一段的工具負責降低「看得懂」的門檻。

### 沉浸式翻譯

<Image src={immersiveTranslate} alt="沉浸式翻譯" />

我最常用的網頁翻譯擴充，可以在保留原文的情況下，把翻譯結果以雙語對照的方式顯示在下方。閱讀外文文章的時候非常好用，不會像 Google 翻譯那樣整頁替換掉原文。

## 收藏與筆記

第三段：讓有用的內容留下來。看完的內容如果沒存下來，大概兩三天就會淡掉；存下來了如果沒再回去看，就跟沒存一樣。這一段的工具處理擷取跟典藏——擷取用 Notion 官方跟第三方擴充快速丟進去，典藏在 Notion app 裡長期保存，再用 Notion Boost 讓回頭翻閱變得舒服。

<Image src={saveToNotion} alt="Save to Notion" />

### Notion

<Image src={notion} alt="Notion" />

Notion 本身是我的知識典藏核心——所有從網頁剪下來的內容、閱讀筆記、研究資料都集中在這裡。重點不是 Notion 多強，而是「所有擷取最終流向同一個地方」這件事。以前試過 Pocket、Instapaper、Apple Notes 各擺一部分，要找的時候都不記得存去哪；現在統一進 Notion，搜尋、分類、雙向連結都可以串起來。

我會在 Notion 裡用資料庫區分「待讀」「已讀有筆記」「參考資料」不同階段，配合 tags 做主題分類。擴充丟進來之後，我會週期性挑出值得寫筆記的條目深化成長期知識，其餘則留作可查詢資料。

<span class="pricing-tag freemium">免費使用，進階功能需訂閱</span>
<a href="https://notion.so" target="_blank" class="official-link">官方網站</a>

<Image src={notionBoost} alt="Notion Boost" />

## 收束

網頁閱讀之所以容易「看完就忘」，不是因為讀得不夠多，而是因為「訊號進來」到「知識留下」這條流程沒被串起來。擷取段決定什麼值得讀、閱讀段讓讀的當下門檻夠低、收藏段讓有用的內容留得下來——三段缺一段，整條流就卡住。工具只是執行器，先建立流程，再配工具。
